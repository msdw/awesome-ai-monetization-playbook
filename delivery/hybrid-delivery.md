# Hybrid Delivery

Hybrid delivery combines elements of multiple delivery models in a single engagement. Common combinations include a project phase followed by ongoing retainer support, or a workshop combined with async deliverables.

## Common hybrid delivery combinations

**Project + retainer**: Build an AI system (project), then maintain and improve it (retainer). The project creates the asset; the retainer protects its value over time.

**Workshop + async delivery**: Run an intensive workshop to define strategy and approach, then deliver the implementation asynchronously over the following weeks.

**Training + DFY**: Train the client's team on AI workflows, then continue delivering the same outputs while the team builds capacity to take over.

**Audit + implementation**: Deliver a diagnostic assessment, then implement the top recommendations from the audit.

## Designing hybrid delivery

When designing a hybrid engagement:
1. Define each phase clearly with its own scope, timeline, and pricing
2. Create explicit transition points between phases — the project ends, the retainer begins
3. Ensure each phase delivers standalone value so the client can stop after any phase
4. Price each phase separately, even if sold together in a bundle

## Transition management

The most critical moment in hybrid delivery is the phase transition. Common failures:
- Project deliverables are incomplete when the retainer starts
- Scope expectations from the project bleed into the retainer
- No formal handoff or sign-off before transitioning

Fix: require explicit client sign-off on each phase before beginning the next.

## Pros

- Matches natural buyer lifecycle (try → build → maintain)
- Creates long-term relationships from single initial sales conversations
- Each phase naturally leads to the next, reducing re-selling effort

## Cons

- More complex to price, scope, and manage than single-model delivery
- Phase transitions require clear documentation and formal agreements
- Clients may try to combine phases to reduce total cost

## When to use

Hybrid delivery is appropriate for complex AI implementations that have both a build phase and ongoing operational requirements. It is particularly effective for AI systems (chatbots, automations, RAG) that need maintenance after initial deployment.
