# Async Delivery

Async delivery is a model where work is completed and communicated through asynchronous channels — email, shared documents, recorded video, or project management tools — without requiring real-time calls or meetings.

## How to implement

1. Define the deliverable format and communication method upfront during onboarding
2. Create a client intake process (form, shared doc, or checklist) that collects all requirements before work starts
3. Use a shared workspace (Google Drive, Notion, Linear) for deliverable handoff
4. Record short Loom videos to explain deliverables or walk through results — more effective than long written explanations
5. Set response time expectations: "I respond within 24 hours on business days"
6. Define revision request procedures — how and where clients submit feedback

## Pros

- Allows delivery across time zones without scheduling constraints
- Enables higher client-to-provider ratios
- Reduces context-switching from constant availability expectations
- Deliverables are documented automatically

## Cons

- Slower resolution of clarifying questions compared to real-time calls
- Client anxiety can increase without visibility into progress
- Some complex feedback is harder to convey asynchronously
- Relationship-building is slower without real-time interaction

## When to use

Async delivery is appropriate for productized services with well-defined inputs and outputs, where the client does not need to be involved in the delivery process itself. It is less appropriate for complex implementations requiring ongoing decision-making from the client.

## Setup requirements

Before offering async-only delivery:
- Written intake process that captures complete requirements
- Clear turnaround time commitments
- Defined revision rounds and request process
- Status update schedule (e.g., weekly progress update email)
